# 0.7 Conventional Commits

> Модуль про понятные коммиты: чтобы история проекта читалась быстро и профессионально.

## Цель модуля

- Привыкнуть писать осмысленные сообщения коммитов.
- Сформировать единый стиль сообщений.
- Упростить чтение истории для себя и для других.

## Теория


### Ключевые концепции

- Концепция 1 — описание
- Концепция 2 — описание

### Базовые типы коммитов

- `feat` — новая функциональность
- `fix` — исправление
- `docs` — документация
- `refactor` — переработка без новой функциональности
- `test` — тесты
- `chore` — технические изменения


### Понимание своими словами

> Основная идея одним абзацем — своими словами, без копипаста.

## Практика

### Задания

- [ ] Переписать 10 примеров плохих commit message в нормальный вид.
- [ ] Сделать минимум 5 коммитов в учебном репозитории по правилам conventional commits.
- [ ] Придумать свой мини-стандарт для учебных проектов.

## Примеры

```text
feat: add initial project structure
docs: update readme with setup steps
fix: correct typo in project title
chore: add gitignore for python project
```

## Артефакты

- Репозиторий с нормальной историей коммитов.
- Краткая памятка по commit message.
