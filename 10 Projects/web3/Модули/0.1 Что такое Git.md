# 0.1 Что такое Git

> Модуль про основу: зачем нужен Git, что такое репозиторий, рабочая директория, staging area и commit.

## Цель модуля

- Понять роль Git в разработке.
- Разобраться в модели хранения изменений.
- Научиться объяснять цикл изменения файла своими словами.

## Теория

### Ключевые концепции

- Репозиторий — место, где Git хранит историю проекта.
- Working directory — текущие файлы, с которыми я работаю.
- Staging area — промежуточная зона перед коммитом.
- Commit — зафиксированное состояние изменений.

### Понимание своими словами

> Git — это система контроля версий, которая помогает сохранять историю изменений проекта, возвращаться к прошлым версиям и безопасно работать над кодом.
