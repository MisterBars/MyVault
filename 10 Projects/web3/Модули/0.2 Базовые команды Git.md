# 0.2 Базовые команды Git

> Модуль про минимальный набор команд, без которого нельзя двигаться дальше: init, status, add, commit, log.

## Цель модуля

- Освоить базовый цикл работы с репозиторием.
- Научиться фиксировать изменения осознанно.
- Привыкнуть смотреть историю и статус проекта.

## Теория

### Команды

- `git init`
- `git status`
- `git add`
- `git commit`
- `git log`

### Понимание своими словами

> Базовый цикл Git выглядит так: создать репозиторий, посмотреть изменения, добавить нужные файлы в staging area, сделать commit и потом проверить историю.

## Практика

### Задания

- [ ] Создать папку `git-basics-demo`.
- [ ] Инициализировать репозиторий через `git init`.
- [ ] Создать файл `README.md`.
- [ ] Проверить изменения через `git status`.
- [ ] Добавить файл через `git add README.md`.
- [ ] Сделать первый коммит.
- [ ] Изменить README ещё 2 раза и сделать ещё 2 коммита.
- [ ] Посмотреть историю через `git log --oneline`.

## Команды

```bash
git init
git status
git add README.md
git commit -m "init: add project readme"
git log --oneline
```

## Артефакты

- Репозиторий с минимум 3 коммитами.
- Скрин или заметка с выводом `git log --oneline`.
