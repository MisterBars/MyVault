# 0.4 Ветки и merge conflicts

> Модуль про ветки: branch, checkout/switch, merge и разрешение конфликтов.

## Цель модуля

- Понять, зачем нужны ветки.
- Научиться безопасно вести параллельную разработку.
- Уметь руками решать простой merge conflict.

## Теория

### Ключевые идеи

- Ветка — отдельная линия разработки.
- `main` — основная ветка проекта.
- Feature-ветки нужны для изоляции изменений.
- Конфликт возникает, когда Git не понимает, какую версию строки оставить.

## Практика

### Задания

- [ ] Создать ветку `feature/readme-update`.
- [ ] Изменить README в `main`.
- [ ] Изменить ту же строку в feature-ветке.
- [ ] Выполнить merge и получить конфликт.
- [ ] Разрешить конфликт вручную.
- [ ] Сделать коммит после разрешения конфликта.

## Команды

```bash
git branch
git switch -c feature/readme-update
git switch main
git merge feature/readme-update
```

## Артефакты

- История с feature-веткой.
- Один вручную решённый merge conflict.

> Где возникло непонимание, как лучше мыслить при работе с ветками.
