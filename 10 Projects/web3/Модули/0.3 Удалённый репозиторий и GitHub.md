# 0.3 Удалённый репозиторий и GitHub

> Модуль про связь локального Git с GitHub: remote, push, pull, clone.

## Цель модуля

- Понять разницу между локальным и удалённым репозиторием.
- Научиться отправлять изменения на GitHub.
- Освоить базовую синхронизацию между устройствами и папками.

## Теория

### Ключевые команды

- `git remote add origin`
- `git push`
- `git pull`
- `git clone`

### Понимание своими словами

> Локальный репозиторий живёт на моём компьютере, а удалённый — на GitHub; Git нужен для истории, а GitHub — для хранения, обмена и совместной работы.

## Практика

### Задания

- [ ] Создать новый учебный репозиторий на GitHub.
- [ ] Привязать локальный репозиторий через `git remote add origin`.
- [ ] Отправить коммиты в GitHub через `git push`.
- [ ] Склонировать тот же репозиторий в другую папку через `git clone`.
- [ ] Сделать изменение в одной папке и подтянуть его в другой через `git pull`.

## Команды

```bash
git remote add origin <repo-url>
git branch -M main
git push -u origin main
git clone <repo-url>
git pull
```

## Артефакты

- GitHub-репозиторий с базовой историей коммитов.
- Две локальные копии одного проекта для проверки clone/pull.

## Заметки и выводы

> Что такое origin, зачем нужен `-u`, чем отличается clone от init.
