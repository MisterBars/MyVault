# 0.5 rebase stash cherry-pick

> Модуль про продвинутые операции: перенос коммитов, временное сохранение незавершённой работы и выборочное применение изменений.

## Цель модуля

- Разобраться, когда использовать rebase, stash и cherry-pick.
- Научиться не бояться переписывания локальной истории.
- Понять разницу между merge и rebase.

## Теория

### Ключевые идеи

- `rebase` переносит ветку на новую базу.
- `stash` временно убирает незакоммиченные изменения.
- `cherry-pick` переносит конкретный коммит в другую ветку.

## Практика

### Задания

- [ ] Создать ветку `feature/rebase-test`.
- [ ] Сделать коммит в `main`.
- [ ] Сделать 2 коммита в feature-ветке.
- [ ] Перебазировать feature-ветку на `main`.
- [ ] Создать незавершённые изменения и убрать их в stash.
- [ ] Вернуть изменения из stash.
- [ ] Сделать отдельный коммит в одной ветке и перенести его через cherry-pick в другую.

## Команды

```bash
git rebase main
git stash
git stash list
git stash pop
git cherry-pick <commit-hash>
```

## Артефакты

- Мини-репозиторий с демонстрацией всех трёх операций.
- Краткая заметка: когда использовать merge, а когда rebase.
