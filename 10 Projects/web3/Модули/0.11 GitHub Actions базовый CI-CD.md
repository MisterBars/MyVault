# 0.11 GitHub Actions базовый CI-CD

> Модуль про первый автоматический workflow: чтобы при push или pull request запускалась проверка проекта.

## Цель модуля

- Понять, что такое CI/CD на самом базовом уровне.
- Создать первый workflow в GitHub Actions.
- Увидеть, как GitHub автоматически проверяет репозиторий.

## Теория


### Ключевые концепции

- Концепция 1 — описание
- Концепция 2 — описание

### Ключевые идеи

- Workflow — сценарий автоматизации.
- Trigger — событие запуска (`push`, `pull_request`).
- Job — отдельная задача внутри workflow.
- Step — шаг внутри job.


### Понимание своими словами

> Основная идея одним абзацем — своими словами, без копипаста.

## Пример workflow

```yaml
name: CI

on:
  push:
    branches: [ main ]
  pull_request:
    branches: [ main ]

jobs:
  basic-check:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout repository
        uses: actions/checkout@v4

      - name: Run basic check
        run: echo "Workflow works"
```
