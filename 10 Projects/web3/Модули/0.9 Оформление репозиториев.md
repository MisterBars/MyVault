# 0.10 Pull Requests Issues Code Review

> Модуль про командную модель работы: задачи, ветки, pull request и проверка изменений.

## Цель модуля

- Понять рабочий цикл через Issue → Branch → PR → Review → Merge.
- Освоить Pull Requests даже в одиночных учебных проектах.
- Сделать процесс разработки более “боевым”.

## Практика

### Задания

- [ ] Создать Issue с описанием небольшой задачи.
- [ ] Создать ветку под задачу.
- [ ] Сделать 1–3 коммита.
- [ ] Открыть Pull Request в своём репозитории.
- [ ] Описать изменения внутри PR.
- [ ] Самому провести mini code review по чеклисту.
- [ ] Выполнить merge.

## Чеклист code review

- Код решает задачу.
- Нет лишних файлов.
- Commit message понятные.
- README не сломан.
- Изменения можно объяснить в 2–3 предложениях.

## Артефакты

- Один Issue.
- Один Pull Request.
- Один merge через PR.

## Заметки и выводы

> Что даёт PR даже при одиночной работе и зачем это сохранять как привычку.
