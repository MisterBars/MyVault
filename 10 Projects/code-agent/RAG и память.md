# RAG и память

## Зачем разделять

В проекте нужно разделять:
- retrieval по документам
- память беседы
- проектную память
- общую базу знаний

Это разные сущности и их нельзя смешивать в одну кучу.

## Слои памяти

### 1. Conversation history
История конкретной беседы:
- сообщения пользователя
- ответы planner/worker
- служебные события
- метаданные сессии

### 2. Global knowledge
Общие знания:
- общие справочные документы
- общие технические заметки
- универсальные шаблоны и паттерны

### 3. Project knowledge
Знания по конкретному проекту:
- архитектура проекта
- файлы проекта
- ключевые решения
- загруженные документы
- полезные веб-страницы по проекту

## Первая версия

На старте достаточно:
- ручной загрузки файлов
- индексации текста
- retrieval по проекту
- простого разделения global / project

## Дальше

Позже можно добавить:
- автоматическое пополнение project memory
- сохранение полезных результатов работы агента
- извлечение знаний из web
