# Веб-интерфейс

## Цель

Сделать собственный интерфейс для общения с агентом, просмотра истории и работы с проектами.

## Базовые функции

Первая версия интерфейса должна уметь:
- показывать список бесед
- открывать конкретную беседу
- отправлять сообщения
- отображать ответы planner и worker
- хранить историю
- переключаться между проектами

## Что важно показывать

Интерфейс должен отображать не только финальный ответ, но и структуру процесса:
- кто ответил: planner или worker
- была ли перепланировка
- использовался ли retrieval
- использовались ли tools
