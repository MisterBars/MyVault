# RPG правила

## Основная идея

Система работает автоматически:

- каждая заметка с `type: task` и `status: done` даёт опыт;
- каждая заметка с `type: project` и `status: done` даёт бонусный опыт через `reward_xp`;
- уровень персонажа считается автоматически по формуле от общего опыта;
- уровень навыка считается автоматически по формуле от опыта задач с нужным `skill`;
- новые навыки подхватываются автоматически, если у них есть `type: skill` и `skill_id`.

## Где ищутся данные

Система ищет заметки по всему vault.

Из расчётов исключаются:
- `90 Templates`
- `40 Archives`

Это значит:
- проекты могут лежать в любых папках;
- задачи могут лежать в любых папках;
- навыки могут лежать в любых папках, но рекомендуется хранить их в `02 Profile/Skills`.

## Типы сущностей

### Проект

```yaml
***
type: project
status: active
deadline: 2026-05-20
reward_xp: 100
tags:
  - project
  - skill/python
***
```

### Задача

```yaml
***
type: task
status: active
skill: python
task_type: normal
project: [[Название проекта]]
deadline: 2026-05-20
tags:
  - task
  - skill/python
***
```

### Навык

```yaml
***
type: skill
skill_id: python
title: Python
tags:
  - skill/python
***
```

## Навыки

Допустимые значения поля `skill` задаёшь сам.

Примеры:
- `python`
- `sql`
- `linux`
- `docker`
- `vba`
- `ai`
- `fitness`
- `learning`
- `postgresql`
- `web`

Главное правило:
- у заметки навыка должен быть `skill_id`;
- у задачи поле `skill` должно точно совпадать с этим `skill_id`.

Пример:

Навык:
```yaml
skill_id: postgresql
```

Задача:
```yaml
skill: postgresql
```

Если ты создашь новую заметку навыка по шаблону и начнёшь использовать такой же `skill` в задачах, этот навык автоматически появится в таблицах и расчётах.

## Типы задач и XP

Если у задачи есть поле `xp`, используется именно оно.

Если `xp` не указано, опыт считается по `task_type`.

| task_type | XP |
|---|---:|
| simple | 10 |
| normal | 20 |
| major | 40 |
| boss | 100 |

Примеры:

```yaml
task_type: simple
```

```yaml
task_type: normal
```

```yaml
task_type: boss
```

Пример задачи с ручным XP:

```yaml
***
type: task
status: done
skill: python
xp: 75
task_type: major
***
```

В этом случае будет использовано `xp: 75`, а не значение из `task_type`.

## Опыт за проекты

Если проект имеет:

```yaml
status: done
reward_xp: 150
```

то 150 XP добавляется в общий опыт персонажа.

Рекомендуется:
- основной опыт получать из задач;
- `reward_xp` использовать как бонус за завершение проекта.

## Формула уровня персонажа

Уровень персонажа считается по общему XP по формуле:

\( level = \lfloor \sqrt{XP / 100} \rfloor + 1 \)

Примеры:
- 0 XP → 1 уровень
- 100 XP → 2 уровень
- 400 XP → 3 уровень
- 900 XP → 4 уровень
- 1600 XP → 5 уровень

Преимущество этой схемы в том, что код не нужно переписывать, когда опыт растёт дальше.

## Формула уровня навыка

Уровень навыка считается по XP навыка по формуле:

\( skillLevel = \lfloor \sqrt{XP / 50} \rfloor + 1 \)

Это делает прокачку навыков чуть быстрее, чем общий рост персонажа.

Примеры:
- 0 XP → 1 уровень
- 50 XP → 2 уровень
- 200 XP → 3 уровень
- 450 XP → 4 уровень
- 800 XP → 5 уровень

## Как работает прогресс внутри уровня

Для персонажа:
- нижняя граница текущего уровня считается автоматически;
- верхняя граница следующего уровня считается автоматически;
- прогресс-бар показывает заполнение между ними.

Для навыка:
- прогресс-бар строится по тем же правилам, но на основе XP конкретного навыка.

## Как завершать записи

### Задача

Было:

```yaml
status: active
```

Стало:

```yaml
status: done
```

### Проект

Было:

```yaml
status: active
```

Стало:

```yaml
status: done
```

После этого:
- XP пересчитывается автоматически;
- уровень пересчитывается автоматически;
- Home и Персонаж обновляются автоматически.

## Как добавлять новый навык

1. Создаёшь новую заметку навыка.
2. Добавляешь в неё `type: skill`.
3. Указываешь уникальный `skill_id`.
4. В задачах начинаешь использовать такой же `skill`.

Пример новой заметки навыка:

```yaml
***
type: skill
skill_id: postgresql
title: PostgreSQL
tags:
  - skill/postgresql
***
```

После этого задачи с:

```yaml
skill: postgresql
```

начнут автоматически влиять на таблицы и расчёты.

## Рекомендации по использованию

- Для обычных действий создавай заметки-задачи.
- Для больших направлений создавай заметки-проекты.
- Если задача относится к проекту, указывай:

```yaml
project: [[Название проекта]]
```

- Для типовых задач используй `task_type`.
- Для особых задач с нестандартной наградой указывай `xp` вручную.
- Не храни `xp_total`, `level` и `skill level` вручную в YAML — они считаются автоматически.

## Рекомендуемые соглашения

### Для тегов

В YAML теги нужно писать без `#`, потому что это корректный YAML-формат.[web:222][web:233]

Правильно:

```yaml
tags:
  - task
  - skill/python
```

В тексте заметки можно писать с `#`:

```markdown
#skill/python
```

### Для статусов

Используй единообразно:
- `active`
- `done`

Позже можно добавить:
- `paused`
- `cancelled`

Но в расчётах RPG участвуют только записи со `status: done`.

## Что влияет на расчёты

В расчёт входят только заметки:
- с корректным `type`;
- с корректным `status`;
- не находящиеся в `90 Templates`;
- не находящиеся в `40 Archives`.

Для навыков дополнительно важно:
- чтобы у навыка был `skill_id`;
- чтобы у задачи `skill` совпадал с `skill_id`.

## Что делать, если запись не отображается

Проверить:
- есть ли YAML в начале файла;
- указан ли `type`;
- указан ли `status`;
- для задач указан ли `skill`;
- для навыков указан ли `skill_id`;
- совпадает ли `skill` задачи со `skill_id` навыка;
- не лежит ли файл в `90 Templates` или `40 Archives`.

## Практический принцип

Система должна быть простой:

- создал задачу;
- указал навык;
- выбрал тип задачи;
- завершил задачу;
- XP и уровень посчитались сами.

Чем меньше ручного редактирования, тем стабильнее и полезнее система.
