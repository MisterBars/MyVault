# Таблица `Users`

## Назначение
- Что хранит таблица.
- Почему она нужна в проекте.
- Кто её использует: формы, модули, процессы.

## Бизнес-смысл
- Что является одной записью таблицы.
- Какую сущность реального мира она описывает.
- В каких сценариях создаётся и обновляется запись.

## Поля
| Поле | Тип | Обяз. | PK | FK | Default | Индекс | Описание |
| --- | --- | --- | --- | --- | --- | --- | --- |
| UserID | AutoNumber | Да | Да | Нет |  | PK | Идентификатор пользователя |
| Login | Text(50) | Да | Нет | Нет |  | Unique | Логин |
| FullName | Text(200) | Да | Нет | Нет |  | No | ФИО |
| RoleID | Long | Да | Нет | Да -> Roles.RoleID |  | Yes | Роль |
| PasswordHash | Text(255) | Да | Нет | Нет |  | No | Хэш пароля |

## Первичный ключ
- `UserID`

## Внешние ключи
| Поле | Ссылка | Обязательность | Что означает |
| --- | --- | --- | --- |
| RoleID | `Roles.RoleID` | Да | Роль пользователя |
| CreatedByUserID | `Users.UserID` | Нет | Кто создал пользователя |

## Уникальные ограничения
- `Login`

## Индексы
| Имя индекса | Поля | Unique | Зачем нужен |
| --- | --- | --- | --- |
| IDXUsersLogin | Login | Да | Быстрый поиск по логину |

## Типовые запросы
```sql
SELECT *
FROM Users
WHERE IsActive = True;
```

```sql
SELECT U.UserID, U.Login, R.RoleName
FROM Users AS U
INNER JOIN Roles AS R ON U.RoleID = R.RoleID;
```
