# RPG правила

## Основная идея

Система работает автоматически:
- каждая заметка с `type: task` и `status: done` даёт опыт;
- каждая заметка с `type: project` и `status: done` даёт бонусный опыт через `reward_xp`;
- уровень персонажа считается по сумме всего опыта;
- уровень навыка считается по опыту задач с нужным `skill`.

## Где ищутся данные

Система ищет заметки по всему vault.
Из расчётов исключаются:
- `90 Templates`
- `40 Archives`

## Типы сущностей

### Проект
```yaml
***
type: project
status: active
deadline: 2026-05-20
reward_xp: 100
tags:
  - project
  - skill/python
***
```

### Задача
```yaml
***
type: task
status: active
skill: python
task_type: normal
project: [[Название проекта]]
deadline: 2026-05-20
tags:
  - task
  - skill/python
***
```

### Навык
```yaml
***
type: skill
skill_id: python
title: Python
tags:
  - skill/python
***
```

## Навыки

Допустимые `skill`:
- `python`
- `sql`
- `linux`
- `docker`
- `vba`
- `ai`
- `fitness`
- `learning`

Поле `skill` у задачи должно совпадать со `skill_id` у навыка.

## Типы задач и XP

Если у задачи указано поле `xp`, берётся оно.
Если `xp` не указано, опыт считается по `task_type`.

| task_type | XP |
|---|---:|
| simple | 10 |
| normal | 20 |
| major | 40 |
| boss | 100 |

## Опыт за проекты

Если проект имеет:
```yaml
status: done
reward_xp: 150
```

то 150 XP добавляется в общий опыт персонажа.

Рекомендуется использовать `reward_xp` только как бонус за завершение проекта, а основной опыт получать из задач.

## Как завершать записи

### Задача
Было:
```yaml
status: active
```

Стало:
```yaml
status: done
```

### Проект
Было:
```yaml
status: active
```

Стало:
```yaml
status: done
```

После этого XP и уровни пересчитываются автоматически в `Home.md` и `Персонаж.md`.

## Рекомендации по использованию

- Для обычных действий создавай заметки-задачи.
- Для больших направлений создавай заметки-проекты.
- Если задача относится к проекту, указывай:
```yaml
project: [[Название проекта]]
```
- Для одноразовых особых задач можешь вручную указать `xp`.
- Для типовых задач достаточно `task_type`.

## Рекомендуемые соглашения

### Для тегов
В YAML:
- `skill/python`
- `skill/linux`

В тексте:
- `#skill/python`
- `#skill/linux`

### Для статусов
Используй единообразно:
- `active`
- `done`
- при желании позже можно добавить `paused`, но в расчётах пока участвуют только `done`.

## Что влияет на расчёты

В расчёт входят только заметки:
- с корректным `type`
- с корректным `status`
- не находящиеся в `90 Templates`
- не находящиеся в `40 Archives`

Если запись не отображается в системе, сначала нужно проверить:
- есть ли YAML в начале файла;
- указан ли `type`;
- указан ли `status`;
- совпадает ли `skill` со `skill_id`.
